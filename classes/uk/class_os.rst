.. _class_OS:

OS
==

**Успадковує:** :ref:`Object<class_Object>`

Забезпечує доступ до функціональних можливостей операційної системи.

.. rst-class:: classref-introduction-group

Опис
--------

Клас **OS** обмотує найбільш поширені функціональні можливості для спілкування з операційною системою хост, наприклад, відео драйвером, затримками, змінними середовищами, виконання бункерів, командного рядка тощо.

\ **Примітка:** У Godot 4, **OS** функції, пов'язані з управління вікнами, буфером, і TTS були перенесені на :ref:`DisplayServer<class_DisplayServer>` однотон (і :ref:`Window<class_Window>` клас). Функції, пов'язані з часом, були видалені і доступні тільки в класі :ref:`Time<class_Time>`.

.. _class_OS_method_execute:

.. rst-class:: classref-method

:ref:`int<class_int>` **execute**\ (\ path\: :ref:`String<class_String>`, arguments\: :ref:`PackedStringArray<class_PackedStringArray>`, output\: :ref:`Array<class_Array>` = [], read_stderr\: :ref:`bool<class_bool>` = false, open_console\: :ref:`bool<class_bool>` = false\ ) :ref:`🔗<class_OS_method_execute>`

Виконує заданий процес *блокуючим* способом. Файл, указаний у ``path``, має існувати та бути виконуваним. Буде використано роздільну здатність системного шляху. ``arguments`` використовуються в заданому порядку, розділені пробілами та взяті в лапки. 

Якщо надається масив ``output``, повний вихід оболонки процесу додається до ``output`` як один елемент :ref:`String<class_String>`. Якщо ``read_stderr`` має значення ``true``, вихід стандартного потоку помилок також додається до масиву. 

У Windows, якщо ``open_console`` має значення ``true`` і процес є консольною програмою, відкривається нове вікно терміналу. 

Цей метод повертає код виходу команди або ``-1``, якщо процес не виконується. 

\ **Примітка:** Основний потік буде заблоковано до завершення виконання команди. Використовуйте :ref:`Thread<class_Thread>`, щоб створити окремий потік, який не блокуватиме основний потік, або використовуйте :ref:`create_process()<class_OS_method_create_process>`, щоб створити повністю незалежний процес.

    int exitCode = OS.Execute("ls", ["-l", "/tmp"], exit);

.. tabs::

    var exit = []

 .. code-tab:: csharp
 
    Godot.Collections.Array exit = []; 
    OS.Execute("CMD.exe", ["/C", "cd %TEMP% && dir"], exit);

\ **Примітка:** Щоб виконати вбудовану команду PowerShell, укажіть ``powershell.exe`` у ``path``, ``-Command`` як перший аргумент, а потрібну команду як другий аргумент. 

\ **Примітка:** Щоб виконати вбудовану команду оболонки Unix, укажіть назву виконуваного файлу оболонки в ``шлях до параметрів``, ``-c`` як перший аргумент, а потрібну команду як другий аргумент.

.. _class_OS_method_get_cmdline_args:

.. rst-class:: classref-method

:ref:`PackedStringArray<class_PackedStringArray>` **get_cmdline_args**\ (\ ) :ref:`🔗<class_OS_method_get_cmdline_args>`

Returns the command-line arguments passed to the engine, excluding arguments processed by the engine, such as ``--headless`` and ``--fullscreen``.

::

    # Godot has been executed with the following command:
    # godot --headless --verbose --scene my_scene.tscn --custom
    OS.get_cmdline_args() # Returns ["--scene", "my_scene.tscn", "--custom"]

Command-line arguments can be written in any form, including both ``--key value`` and ``--key=value`` forms so they can be properly parsed, as long as custom command-line arguments do not conflict with engine arguments.

You can also incorporate environment variables using the :ref:`get_environment()<class_OS_method_get_environment>` method.

You can set :ref:`ProjectSettings.editor/run/main_run_args<class_ProjectSettings_property_editor/run/main_run_args>` to define command-line arguments to be passed by the editor when running the project.

\ **Example:** Parse command-line arguments into a :ref:`Dictionary<class_Dictionary>` using the ``--key=value`` form for arguments:


.. tabs::

 .. code-tab:: gdscript

    var arguments = {}
    for argument in OS.get_cmdline_args():
        if argument.contains("="):
            var key_value = argument.split("=")
            arguments[key_value[0].trim_prefix("--")] = key_value[1]
        else:
            # Options without an argument will be present in the dictionary,
            # with the value set to an empty string.
            arguments[argument.trim_prefix("--")] = ""

 .. code-tab:: csharp

    var arguments = new Dictionary<string, string>();
    foreach (var argument in OS.GetCmdlineArgs())
    {
        if (argument.Contains('='))
        {
            string[] keyValue = argument.Split("=");
            arguments[keyValue[0].TrimPrefix("--")] = keyValue[1];
        }
        else
        {
            // Options without an argument will be present in the dictionary,
            // with the value set to an empty string.
            arguments[argument.TrimPrefix("--")] = "";
        }
    }



\ **Note:** Passing custom user arguments directly is not recommended, as the engine may discard or modify them. Instead, pass the standard UNIX double dash (``--``) and then the custom arguments, which the engine will ignore by design. These can be read via :ref:`get_cmdline_user_args()<class_OS_method_get_cmdline_user_args>`.

- ``VARIANT`` - варіант мови, регіон і сорт замовлення. Варіант може мати будь-яку кількість зашифрованих слів.

- ``extra`` - напівколон відокремлений список додаткових ключових слів. Це може включати валюту, календар, порядок сортування та інформацію про систему.

- ``"access"`` - кількість пам'яті, яка може бути виділена без розширення файлу swap (s), в байтах. Це значення включає як фізичну пам'ять, так і ковпачок.

- ``"stack"`` - розмір поточного стека ниток в байтах.

\ **Note:** Кожне значення запису може бути ``-1``, якщо це невідомо.

:ref:`PackedByteArray<class_PackedByteArray>` **read_buffer_from_stdin**\ (\ buffer_size\: :ref:`int<class_int>` = 1024\ ) :ref:`🔗<class_OS_method_read_buffer_from_stdin>`

Зчитує введені користувачем дані як необроблені дані зі стандартного вводу. Ця операція може бути *блокуючою*, що призводить до зависання вікна, якщо :ref:`read_buffer_from_stdin()<class_OS_method_read_buffer_from_stdin>` викликається в головному потоці.

Якщо стандартний ввід є консольним, виклик цього методу без обгортки консолі призведе до остаточного зависання. Якщо стандартний ввід є каналом або файлом, його можна використовувати без обгортки консолі. Якщо вам потрібен один виконуваний файл з повною підтримкою консолі, використовуйте власну збірку, скомпільовану з прапорцем ``windows_subsystem=console``.

.. _class_OS_method_shell_open:

.. rst-class:: classref-method

:ref:`Error<enum_@GlobalScope_Error>` **shell_open**\ (\ uri\: :ref:`String<class_String>`\ ) :ref:`🔗<class_OS_method_shell_open>`

Запитує ОС відкрити ресурс, визначений ``uri``, за допомогою найбільш підходящої програми. Наприклад: 

- ``OS.shell_open("C:\\Users\\name\\Downloads")`` у Windows відкриває файловий провідник у папці Downloads користувача. 

- ``OS.shell_open("C:/Users/name/Downloads")`` також працює в Windows і відкриває файловий провідник у папці "Завантаження" користувача. 

- ``OS.shell_open("https://godotengine.org")`` відкриває веб-переглядач за умовчанням на офіційному веб-сайті Godot. 

- ``OS.shell_open("mailto:example@example.com")`` відкриває поштовий клієнт за умовчанням із полем "Кому" встановлено значення ``example@example.com``. Перегляньте `RFC 2368 – [code]mailto[/code] URL-схема <https://datatracker.ietf.org/doc/html/rfc2368>`__, щоб переглянути список полів, які можна додати. 

Використовуйте :ref:`ProjectSettings.globalize_path()<class_ProjectSettings_method_globalize_path>`, щоб перетворити шлях до проекту ``res://`` або ``user://`` у системний шлях для використання з цим методом. 

\ **Примітка.** Використовуйте :ref:`String.uri_encode()<class_String_method_uri_encode>`, щоб кодувати символи в URL-адресах безпечним для URL-адреси переносним способом. Це особливо потрібно для розривів рядків. Інакше :ref:`shell_open()<class_OS_method_shell_open>` може не працювати належним чином у проекті, експортованому на веб-платформу. 

\ **Примітка.** Цей метод реалізовано на Android, iOS, Web, Linux, macOS і Windows.

:ref:`Error<enum_@GlobalScope_Error>` **shell_show_in_file_manager**\ (\ file_or_dir_path\: :ref:`String<class_String>`, open_folder\: :ref:`bool<class_bool>` = true\ ) :ref:`🔗<class_OS_method_shell_show_in_file_manager>`

Запити OS для відкриття файлового менеджера, навігація на даній ``file_or_dir_path`` і виберіть цільовий файл або папку.

Якщо ``open_folder`` є ``true`` і ``file_or_dir_path`` є дією довідковою доріжкою, ОС відкриє файловий менеджер і навігувати на цільову папку без вибору нічого.

Використовуйте :ref:`ProjectSettings.globalize_path()<class_ProjectSettings_method_globalize_path>` для перетворення ``res://`` або ``user://`` шлях проекту в шлях системи, щоб використовувати цей метод.

\ **Примітка:** Цей метод реалізований тільки на Windows і macOS. На інших платформах вона повернеться до :ref:`shell_open()<class_OS_method_shell_open>` з довідковою доріжкою ``file_or_dir_path``, зафіксовано ``файлом ``.

.. _class_OS_method_unset_environment:

.. rst-class:: classref-method

|void| **unset_environment**\ (\ variable\: :ref:`String<class_String>`\ ) |const| :ref:`🔗<class_OS_method_unset_environment>`

Видаліть даній зміні середовища з поточного середовища, якщо вона існує. Ім'я ``variable`` не може бути порожнім або включати символ ``=``. Природна змінна буде видалена для процесу Godot і будь-якого процесу, виконаного з :ref:`execute()<class_OS_method_execute>` після запуску :ref:`unset_environment()<class_OS_method_unset_environment>`. Видалення змінної середовища буде * не*, щоб процеси, які працюють після завершення процесу Godot.

\ **Примітка:** Внутрішня змінна імена є чутливими на всіх платформах, крім Windows.
