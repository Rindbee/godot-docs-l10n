Tree
====

**Успадковує:** :ref:`Control<class_Control>` **<** :ref:`CanvasItem<class_CanvasItem>` **<** :ref:`Node<class_Node>` **<** :ref:`Object<class_Object>`

Контроль, який використовується для відображення комплексу внутрішніх :ref:`TreeItem<class_TreeItem>` в ієрархічній структурі.

.. rst-class:: classref-introduction-group

Опис
--------

Елемент керування, який використовується для відображення набору внутрішніх :ref:`TreeItem<class_TreeItem>` в ієрархічній структурі. Елементи дерева можна вибирати, розгортати та згортати. Дерево може мати кілька стовпців із настроюваними елементами керування, такими як :ref:`LineEdit<class_LineEdit>`, кнопки та спливаючі вікна. Це може бути корисним для структурованих показів і взаємодії. 

Дерева будуються за допомогою коду з використанням об’єктів :ref:`TreeItem<class_TreeItem>` для створення структури. Вони мають один корінь, але кілька коренів можна симулювати за допомогою :ref:`hide_root<class_Tree_property_hide_root>`: 


.. tabs:: 

 .. code-tab:: gdscript
 
    func _ready(): 
        var tree = Tree.new() 
        var root = tree.create_item() 
        tree.hide_root = ture 
        var child1 = tree.create_item(root) 
        var child2 = tree.create_item(root) 
        var subchild1 = tree.create_item(child1) 
        subchild1.set_text(0, "Subchild1")  

 .. code-tab:: csharp
 
    public override void _Ready() 
    {
        var tree = new Tree(); 
        Корінь TreeItem = tree.CreateItem(); 
        tree.HideRoot = true; 
        TreeItem child1 = tree.CreateItem(root); 
        TreeItem child2 = tree.CreateItem(root); 
        TreeItem subchild1 = tree.CreateItem(child1); 
        subchild1.SetText(0, "Subchild1"); 
    } 

 

Щоб виконати ітерацію по всіх об’єктах :ref:`TreeItem<class_TreeItem>` в об’єкті **Tree**, використовуйте :ref:`TreeItem.get_next()<class_TreeItem_method_get_next>` і :ref:`TreeItem.get_first_child()<class_TreeItem_method_get_first_child>` після отримання кореня через :ref:`get_root()<class_Tree_method_get_root>`. Ви можете використовувати :ref:`Object.free()<class_Object_method_free>` для :ref:`TreeItem<class_TreeItem>`, щоб видалити його з **Tree**. 

\ **Поступовий пошук:** Подібно до :ref:`ItemList<class_ItemList>` і :ref:`PopupMenu<class_PopupMenu>`, **Tree** підтримує пошук у списку, коли елемент керування знаходиться в фокусі. Натисніть клавішу, яка відповідає першій літері назви об’єкта, щоб вибрати перший об’єкт, який починається з заданої літери. Після цього є два способи виконати поступовий пошук: 1) Натисніть ту саму клавішу ще раз до закінчення часу очікування, щоб вибрати наступний елемент, який починається з тієї ж літери. 2) Натискайте клавіші з літерами, які відповідають решті слова перед відповідним часом очікування, щоб безпосередньо вибрати відповідний елемент. Обидві ці дії буде скинуто на початок списку, якщо час очікування минув після останнього натискання клавіші. Ви можете налаштувати тривалість тайм-ауту, змінивши :ref:`ProjectSettings.gui/timers/incremental_search_max_interval_msec<class_ProjectSettings_property_gui/timers/incremental_search_max_interval_msec>`.

.. _enum_Tree_SelectMode:

.. rst-class:: classref-enumeration

enum **SelectMode**: :ref:`🔗<enum_Tree_SelectMode>`

.. _class_Tree_constant_SELECT_SINGLE:

.. rst-class:: classref-enumeration-constant

:ref:`SelectMode<enum_Tree_SelectMode>` **SELECT_SINGLE** = ``0``

Дозволяє підбір однієї комірки одночасно. З точки зору елементів можна вибрати тільки один елемент. І є тільки один стовпчик, вибраний в вибраному елементі.

Курсор фокусу завжди прихований в цьому режимі, але він розташований на поточному підборі, що робить в даний час вибраний елемент на даний момент фокусований елемент.

.. _class_Tree_constant_SELECT_ROW:

.. rst-class:: classref-enumeration-constant

:ref:`SelectMode<enum_Tree_SelectMode>` **SELECT_ROW** = ``1``

Дозволяє підбір одного ряду одночасно. З точки зору елементів можна вибрати тільки одинарні елементи. І всі стовпчики вибирають у вибраному елементі.

Курсор фокусу завжди прихований в цьому режимі, але він розташовується в першому стовпчику поточного вибору, що робить в даний час обраний елемент на даний момент фокусований елемент.

.. _class_Tree_constant_SELECT_MULTI:

.. rst-class:: classref-enumeration-constant

:ref:`SelectMode<enum_Tree_SelectMode>` **SELECT_MULTI** = ``2``

Дозволяє вибрати декілька клітин одночасно. З точки зору елементів можна вибрати декілька елементів. І можуть бути декілька стовпців, вибраних в кожному вибраному пункті.

Курсор фокусу видно в цьому режимі, елемент або стовпець під курсор не обов'язково вибраний.

Увімкнено "приблизний елемент" та "повільний елемент" розділи крапель. В розділі «приблизний елемент» покривається верхню половину виробу, а розділ «повільний елемент» охоплює нижню половину.
